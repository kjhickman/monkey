- Add module-level function declarations first (fn Foo(...) -> T { ... })
- Then add/require main
- Then disallow top-level let/expressions
A practical roadmap:
- Add a new AST node for function declarations (top-level statement/item), instead of relying on let f = fn....
- Parser: support fn Name(...) -> T { ... } at top level.
- Resolver: predeclare all named functions in module scope first (enables mutual recursion).
- Type checker: two-pass function checking
  - pass 1: register signatures
  - pass 2: validate bodies against signatures
- IR lowering/emitter:
  - lower named functions directly as declared methods
  - keep closure path for function literals (still needed for first-class funcs)
- Entry-point policy:
  - require exactly one fn main() -> int (or void, your call)
  - build/run fail with clear diagnostics if missing/invalid
- Tighten top-level rules:
  - disallow top-level executable statements (let, expression statements)
  - allow only declarations (functions, later maybe types/constants)
- Tests:
  - parser/resolver/typechecker/IR/emitter integration for
    - named calls
    - recursion/mutual recursion
    - main validation failures
    - disallowed top-level code
