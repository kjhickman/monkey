TOP 8 FINDINGS (RANKED BY IMPACT)
================================================================================

1. ⭐⭐⭐⭐⭐ CRITICAL: Hardcoded Builtin Mappings
   Location: IrLowerer.cs:927-984 (TryLowerBuiltinCallName)
             ClrPhase1Executor.cs:927-942 (BuildBuiltinMap)
             ClrRuntimeBuiltins.cs:3-64 (implementations)
   Problem: 3 places, no single source of truth
   Impact: Blocks NuGet interop, test brittleness
   Files Affected: 3 (IrLowerer, ClrPhase1Executor, ClrRuntimeBuiltins)

2. ⭐⭐⭐⭐ CRITICAL: Builtin Type Specialization
   Location: IrLowerer.cs:927-984
   Problem: Monomorphic dispatch (one overload per type)
   Impact: Each new type requires 3+ code changes
   Fix: BuiltinRegistry with type-signature resolution

3. ⭐⭐⭐⭐⭐ HIGH: Type Mapping Hardcoding
   Location: ClrPhase1Executor.cs:641-685 (MapType)
   Problem: No extensibility for custom/BCL types
   Impact: Blocks BCL interop, generic types unsupported
   Fix: Extract ITypeMapper interface

4. ⭐⭐⭐⭐ HIGH: Monolithic IR → CLR Emission
   Location: ClrPhase1Executor.cs:150-256 (EmitAssembly)
             ClrPhase1Executor.cs:258-639 (EmitFunction)
   Problem: Single method orchestrates everything
   Impact: Can't test phases independently, blocks modular compilation
   Fix: Visitor pattern + phase extraction

5. ⭐⭐⭐ MEDIUM: Runtime Builtin Coupling
   Location: ClrRuntimeBuiltins.cs:3-64
   Problem: Hardcoded assumptions (int64 arrays, no context)
   Impact: AOT incompatibility, no extensibility
   Fix: IBuiltinProvider wrapper

6. ⭐⭐⭐ MEDIUM: IR Type Assumptions
   Location: IrLowerer.cs:1118-1132 (IsSupportedRuntimeType)
   Problem: Whitelist hardcoded in lowerer
   Impact: Blocks custom types, modular compilation
   Fix: Move to ITypeMapper

7. ⭐⭐⭐ MEDIUM: Large IR → IL Switch
   Location: ClrPhase1Executor.cs:327-596 (EmitFunction switch)
   Problem: 20+ cases tightly bound to IL emission
   Impact: Blocks optimization passes, debugging, interpretation
   Fix: Visitor pattern

8. ⭐⭐ LOW: Mono.Cecil Tight Coupling
   Location: Throughout ClrPhase1Executor.cs
   Problem: No abstraction over IL generation
   Impact: Backend lock-in (long-term)
   Fix: IL generation abstraction layer
